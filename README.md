# Slay the Spire Like

現在開発中のデッキ構築型ローグライク（Slay the Spireライク）ゲームのプロジェクトです。
画面の描画はプレーンなHTML、CSS、Vanilla TypeScriptを用いて構築され、フレームワークに依存せずViteによるビルド構成を採用しています。

## ⚙️ 技術スタック
- **言語**: TypeScript (5.9.3)
- **ビルドツール**: Vite (7.2.4)
- **スタイル**: Vanilla CSS (`style.css`)
- **レンダリング/操作**: `main.ts` をエントリーポイントとしたDOM更新とCanvasの活用

## 📂 プロジェクト構成（アーキテクチャ）
以下は、中核となるディレクトリとファイルの役割です。

### 📌 `src/` (ソースコード)
- `main.ts`: アプリケーションのエントリーポイント。ゲームループの起動、各種UI（トップバー、デッキ画面、戦闘画面、ショップ画面）の制御やユーザー入力を処理します。
- `style.css`: 全体のUIスタイルやアニメーション定義。

### 📌 `src/core/` (ゲームシステムの中核)
各システムごとに責務を分割し、コンポーネント指向での設計になっています。

- **`engine.ts`**: ゲームエンジン。ターンの進行、引いて捨てるサイクル、戦闘の全体的な状態（勝利・敗北判定など）を管理します。
- **`entity.ts`**: プレイヤー（Player）と敵キャラクター（Enemy）の基底クラス。HP、ブロック、ステータス異常（バフ・デバフ）、ダメージ処理を担います。
- **`status-effect.ts`**: 脆弱、脱力、筋力、敏捷性、毒などの状態異常システムを定義し、ダメージの増減等へ介入するロジックを管理します。
- **`cards/` & `card-class.ts`**: カードシステムのベースと個別カード。アタック、スキル、パワー等、カード種類ごとに効果を定義し、ドロー・廃棄・使用などのアクションを担当します（現在は `cards/` ディレクトリ配下に分割して循環参照を回避しています）。
- **`relics/` & `relic-class.ts`**: レリック（永続アーティファクト）のシステムと個別レリック。戦闘開始時やターン経過、特定カード使用時に発動するパッシブ効果（カウンター等）を処理します（`relics/` フォルダ配下にレアリティ別に分割中）。
- **`enemies/`**: スライム、ラウス、ヤウカオなど、各敵キャラクターのAI（行動パターンと意図の決定機能）。
- **`potion.ts` & `potion-data.ts`**: ポーションシステムの実装。複数種類のポーションの持続やターゲット指定効果を管理します。
- **`map-generator.ts` & `map-data.ts`**: ノードベースでルートが分岐するマップの自動生成ロジック。
- **`encounter-data.ts`**: 各階層にどの敵プールの組み合わせが登場するかを担当するデータベース。
- **`event-data.ts` & `boss-data.ts`**: 「？」マス等で発生するイベントや、ボス戦情報の定義。
- **`audio-manager.ts` & `effect-manager.ts`**: サウンドエフェクト、および画面上の視覚的エフェクト（VFX）の管理。
- **`scene-manager.ts`**: タイトル、マップ、戦闘、イベント等、ゲームシーン間の遷移管理。

### 📌 その他のディレクトリ
- **`scripts/`**: ビルド時に循環参照を解消したり、TypeScriptファイルを分割したりするためのNode.js製ユーティリティスクリプト。
- **`.agent/skills/`**: AIアシスタント（Agent）が開発を支援するための実装手順やベストプラクティス（スキル）が定義されたフォルダ。新しい状態異常やカードを追加する際のお作法が記載されています。
- **`task.md` / `Enemy_Task.md`**: 実装済みの機能や残件、敵AIに関するタスク管理ファイル。現在の進捗状況が一目で把握できます。

## 🚀 現在の主要な進行状況（STATUS）
ゲームとしての根幹部分はすでに連携しており、実際のSlay the Spireのプレイングに近い挙動が再現されています。

1. **基本システムの結合**: 敵（AI）、カードプレイのプレビュー、ダメージ計算（筋力/脆弱/ブロック等）、マナコスト管理、戦闘勝利と報酬獲得（カードやレリック、ゴールド、ポーション）までがスムーズに動作します。
2. **カードとレリックの拡張**: アイアンクラッドのアタック、スキル、パワーカードの多くが実装済み。レリックやポーションについても主要な機能の実装が進んでいます（カウンター、特定効果の永続付与など）。
3. **リファクタリング**: 規模の拡大に伴い、巨大になりがちな `card.ts` や `relic.ts` を分割しつつ循環参照等のバグを解決中で、スケーラビリティが大きく向上しました。
4. **UIの精緻化**: アイコン表示、ツールチップ、ヘッダー（HP、ゴールド情報）、デッキ内容の確認・廃棄などのUI対応が整備されつつあります。

## 🔧 開発向けルール（運用）
このプロジェクトでは、新規追加が必要な場合（例：状態異常追加、カード追加）、`.agent/skills/` または `/workflows` コマンドで定められた手順を用いて追加し、関連ファイル（`task.md` など）のチェックリストを更新しながら進行します。
